# The Invention Archive

A long-term (100+ years), structured archival system designed to catalog, organize, and preserve conceptual constructs, theoretical frameworks, and AI repositories.

## Purpose

The Invention Archive serves as a highly organized, durable repository for 21 core inventions and research frameworks spanning advanced AI architectures, thermodynamic models, cognitive fields, and evolutionary organoids.

It is designed for longevity and resilience, utilizing universally accessible, open-standard formats to ensure data survivability and readability across generations.

## Architecture & Formats

To guarantee long-term accessibility, the archive strictly adheres to the following file formats:
* **Markdown (`.md`)** - For human-readable documentation, logs, and theoretical context.
* **JSON (`.json`)** - For machine-readable metadata and tracking.
* **ZIP (`.zip`)** - For immutable source code snapshots.
* **Text (`.txt`)** - For file trees and basic manifest data.

## Core Structure

* `constructs/` — The core units of the archive. Each conceptual framework or invention is isolated in its own numbered directory (e.g., `001_HRF`, `002_NECF`).
* `docs/` — System-level documentation, daily logs, and tracking metadata.
* `standards/` — Operational protocols defining naming conventions, directory structures, and versioning rules.
* `INDEX.md` — The master registry mapping every construct's ID, Acronym, Type, Status, and internal Path.
* `TIMELINE.md` — A chronological record of construct inception and evolution.
* `RELATIONS.md` — A theoretical map defining the interdependencies and structural connections between various constructs.
* `LEXICON.md` — A unified glossary of terminology used across the frameworks.
